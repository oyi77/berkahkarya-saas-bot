## Changes Made
1. Added import for additional queue instances (`paymentQueue`, `notificationQueue`, `cleanupQueue`)
2. Added `getQueueByName()` helper function to retrieve queue by name
3. Added `/api/queue/retry/:jobId` endpoint - retries a failed job
4. Added `/api/queue/clean` endpoint - cleans completed/failed jobs older than 24h
5. Added `/api/queue/` to admin auth route check

## Implementation Details
- **Retry endpoint**: Uses BullMQ's `job.retry()` method to retry a failed job
- **Clean endpoint**: Uses BullMQ's `queue.clean()` method to remove completed/failed jobs older than specified hours (default 24h)
- Both endpoints require admin authentication (Basic auth or cookie)
- Optional `queue` query/body param to specify which queue (video, payment, notification, cleanup)
