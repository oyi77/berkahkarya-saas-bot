# Task 11: Admin Subscription Management API - Evidence

## Date: 2026-04-02

## Implementation Summary

Added 3 new admin subscription management API endpoints to `src/routes/admin.ts`:

### 1. GET /api/subscriptions/active
- Lists all active subscriptions with user info
- Returns: id, userId, userTelegramId, userUsername, userFirstName, userTier, plan, billingCycle, status, currentPeriodStart, currentPeriodEnd, cancelAtPeriodEnd, createdAt
- Pagination support via limit (default 50, max 200) and offset

### 2. POST /api/subscriptions/:id/cancel
- Cancels a subscription by setting cancelAtPeriodEnd flag
- Validates subscription exists and is active
- Returns success message

### 3. POST /api/subscriptions/:id/extend
- Extends subscription by adding days to currentPeriodEnd
- Accepts optional body.days parameter (default 30, min 1, max 365)
- Validates subscription exists and is active
- Returns newPeriodEnd date

## Authentication
All endpoints are protected by admin auth (Basic auth or admin_token cookie)
